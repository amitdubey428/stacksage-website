FAQ

Where does StackSage run?

Inside your GitHub Actions runner.

How is StackSage delivered?

As a private Docker image pulled from GHCR and run by your workflow.

Do you ingest AWS credentials?

No. AWS access is via a customer-controlled read-only IAM role used by your workflow.

What is the StackSage license?

A signed, time-limited token provided by StackSage. If it expires, the audit fails fast until it’s renewed.

What do I get as output?

An HTML report plus machine-readable findings (JSON/CSV) as workflow artifacts.

What AWS services do you cover?

Common waste & hygiene checks across EC2, EBS (incl. snapshots), RDS, S3, and network resources. Optional enrichments: CloudWatch utilization + Cost Explorer historical spend (opt-in).

Findings are based on what the workflow can read with the permissions you grant; outcomes vary by account and usage.
