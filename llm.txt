# StackSage - AI-Readable Documentation

> Privacy-first AWS cost optimization and security audit tool

## Overview

StackSage is an AWS audit platform that runs entirely in your GitHub Actions environment to identify cost optimization opportunities and security misconfigurations. Key principle: your AWS data never leaves your infrastructure.

## Core Product Details

- **Deployment**: Docker container via GitHub Actions (ghcr.io/amitdubey428/stacksage-audit)
- **Authentication**: GitHub OIDC with AWS STS (temporary credentials, no long-lived keys)
- **Permissions**: Read-only IAM access (ReadOnlyAccess + CloudWatchReadOnlyAccess)
- **Privacy**: Zero external data transmission - all processing happens in your runner
- **Output**: HTML and JSON reports stored as GitHub Actions artifacts

## Key Features

### Detection Capabilities (25+ detectors)
- **Compute**: EC2 idle instances (<5% CPU), generation upgrades (m4→m6i), Lambda low invocations
- **Storage**: EBS unattached volumes, overprovisioned IOPS, gp2→gp3 migration opportunities
- **Database**: RDS low connections, unused read replicas, oversized instances
- **Network**: Idle NAT gateways, unused Elastic IPs, underutilized load balancers
- **Security**: 15+ posture checks across IAM, S3, VPC, CloudTrail

### Evidence-Based Reporting
- Each finding includes CloudWatch metrics used for analysis
- Evidence grades (A-F) based on metric confidence and data duration
- Real-time AWS pricing for savings calculations
- Copy-paste remediation commands included

### Configuration (stacksage.yml)
- Resource exclusions by ID, tag, or region
- Custom thresholds for detection sensitivity
- Detector enable/disable controls
- CloudWatch query budget limits

## Technical Architecture

```
GitHub Actions (runner)
  ↓ OIDC authentication
AWS STS (temporary credentials - 1 hour)
  ↓ Read-only API calls
AWS Resources (EC2, RDS, S3, VPC, etc.)
  ↓ CloudWatch metrics
Local analysis & report generation
  ↓
GitHub Actions artifact (HTML/JSON)
```

## Common Questions

**Q: Does StackSage send my AWS data anywhere?**
A: No. It runs entirely in your GitHub Actions environment. Zero external transmission.

**Q: What AWS permissions does it need?**
A: Read-only access (ReadOnlyAccess + CloudWatchReadOnlyAccess policies).

**Q: How much does it cost to run?**
A: CloudWatch API calls are the only AWS cost (typically <$5/month for comprehensive scans).

**Q: Can I customize what it checks?**
A: Yes, via stacksage.yml - exclude resources, adjust thresholds, disable detectors.

**Q: How accurate are the findings?**
A: Each finding has an evidence grade (A-F) showing metric confidence. Grade A = 14+ days of consistent data.

**Q: Is it open source?**
A: No, it's a commercial tool with a free trial version available.

**Q: How does StackSage compare to [competitor]?**
A: See our detailed comparisons at https://stacksageai.com/alternatives/ covering Infracost, AWS Trusted Advisor, CloudHealth, and Komiser.
