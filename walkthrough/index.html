60‑second walkthrough

StackSage runs inside your GitHub Actions runner, assumes a customer‑controlled read‑only role, and produces an audit pack you can share (summary + HTML) and automate (JSON/CSV).

Open Sample Audit Pack →Start Trial →

1) summary.md

A one-page, CTO-ready summary you can paste into an email, Slack, or a ticket.

# StackSage Audit Summary

- Findings: 17
- Est. monthly savings: $312.40
- CloudWatch metrics: enabled (14d lookback)

Top actions:
1) Delete unused EBS volumes (~$84/mo)
2) Migrate gp2 → gp3 (~$51/mo)
3) Fix open security groups (posture)

2) findings.json / findings.csv

Machine-readable outputs for automation, dashboards, and internal reporting.

[{
  "type": "lambda_graviton_migration",
  "resource_type": "lambda",
  "id": "my-function",
  "severity": "low",
  "confidence": 0.85,
  "estimated_monthly_savings_usd": 12.34,
  "verification_commands": ["aws lambda get-function-configuration ..."]
}]

3) audit_report.html

A shareable HTML report with evidence and copyable verification commands.

audit_report.html
- Executive summary
- Evidence & metrics
- Remediation commands
- Provenance (budget + errors)

What to do next

  • • If you want proof-first: open the sample audit pack and skim the top findings.
  • • If you want to run it: follow the Trial guide and run once inside your repo.
  • • If you want full coverage: use the paid GitHub Workflow request form from the homepage.
